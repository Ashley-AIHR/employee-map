# 第 03 章：搭建前端骨架与页面结构

模块和工作流拆清楚以后，这一章开始真正动手搭产品骨架。

## 本章目标

用尽量简单、但足以承载真实交互的前端结构，把案例的主要页面和状态容器先立起来。

## 这个案例最后采用的实现结构

为了保证：

- 本地运行简单
- 别人容易看懂
- 适合公开学习
- 后续方便快速迭代

这次最后采用的是最直接的一套结构：

- `index.html`：页面挂载点
- `styles.css`：视觉系统与布局
- `script.js`：数据、状态、导入、计算、导出、渲染

## 页面骨架至少要包含什么

在这个案例里，前端骨架至少需要这些区域：

- 左侧导航
- 顶部工具栏
- 主要视图区
- 导入抽屉
- 字段映射抽屉
- 导出区域

导航上最终稳定下来的主视图包括：

- 组织地图
- 人员清单
- 编制与空缺
- 结构信号
- 报表输出
- 规则配置

## 为什么这里不先追求“做得很像”

因为这一步的重点不是还原视觉，而是让系统先有：

- 稳定的视图切换
- 可持续扩展的状态结构
- 能接真实数据的容器

如果这一章就把精力全放在细节样式上，后面接数据时很容易返工。

## 这一章里最重要的实现原则

- 页面结构先完整，再局部精修
- 状态先集中，再逐步细化
- 视图先可切换，再做复杂图表

## 可以直接给 Codex 的指令

```text
请先搭一个前端-only 的应用骨架，包含导航、主视图区、工具栏、抽屉和基础状态容器。后续要能接入真实数据和导入流程。
```

## 本章完成标准

这一章结束时，你应该已经有：

- 可切换的前端骨架
- 明确的主视图区
- 后续能接数据、导入、导出的状态结构

到这里，系统才有地方长内容。
