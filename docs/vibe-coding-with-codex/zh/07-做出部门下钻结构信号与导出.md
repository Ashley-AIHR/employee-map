# 第 07 章：做出部门下钻、结构信号与导出

这一章会把案例真正推进到“产品成立”的层级。  
因为有数据、有导入，还不够。系统必须能让用户切上下文、读判断、拿结果。

## 本章目标

完成三件关键事情：

1. 部门下钻
2. 结构信号
3. 导出结果

## 部门下钻在这个案例里意味着什么

不是改标题，也不是换一张图，而是让当前部门上下文影响整个系统。

当用户切到某个部门时，至少这些内容都应该联动：

- 当前人数
- 新人占比
- 管理跨度
- 职级结构
- 城市分布
- 人员清单
- 编制与空缺
- 结构信号
- 导出结果

## 结构信号在这个案例里怎么生成

当前版本基于前端规则生成信号，主要看：

- 管理跨度是否过大
- 新人占比是否过高
- P6+ 占比是否偏低
- 开放 HC 是否偏多

也就是说，结构信号的作用不是重复图表，而是把图表翻译成判断。

## 导出在这个案例里要导什么

当前案例里，导出不是摆设，而是要能带走当前上下文的结果。至少包括：

- 当前人员 CSV
- 结构信号 CSV
- 摘要文本 / JSON

## 为什么这三件事必须放一起做

因为它们共享同一个核心前提：  
当前部门上下文必须是真的。

如果下钻不真实：

- 信号会不准
- 导出会不准
- 产品体验会变成“看起来很丰富，实际不可信”

## 可以直接给 Codex 的指令

```text
请让部门切换影响所有相关视图，并基于当前上下文生成结构信号与导出结果。不要只更新标题或局部图表。
```

## 本章完成标准

做完这一章，这个案例应该已经具备：

- 真实联动的部门下钻
- 基于规则的结构信号
- 能导出当前上下文结果的能力

到这里，它才真正开始像一个组织分析产品。
