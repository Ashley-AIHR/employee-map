# 第 06 章：实现 CSV 导入与字段映射

到这一章，案例要从“内置 demo 可看”升级成“别人能拿文件亲自试”。

## 本章目标

把 CSV 导入做成一条完整、真实、可重复测试的浏览器工作流。

## 这个案例里的导入流程

一个完整导入流程至少要完成这些步骤：

1. 读取本地 CSV 文件
2. 解析成行
3. 识别表头
4. 映射到标准字段
5. 生成员工对象
6. 执行基础校验
7. 重建部门结构
8. 刷新所有视图
9. 保存到本地状态

## 为什么字段映射是核心能力

因为真实花名册不可能完全统一。

这次案例当前支持的别名包括：

- `employee_id / 员工ID / 员工编号 / id / ID`
- `name / 姓名 / 员工姓名`
- `department / department_l2 / 部门 / 二级部门 / 团队`
- `job_title / 岗位名称 / 岗位 / 职位`
- `contract_type / 合同类型 / 用工类型 / 用工形式`
- `hire_date / 入职日期 / 入职时间`

如果只认单一字段名，这个案例很快就会失去现实可用性。

## 这次案例里配套准备了什么

为了让读者最快测通这条链路，repo 里准备了一份标准文件：

- `outputs/test-employee-map-import.csv`

这样别人进入项目后，可以先用已知成功输入测通系统，再替换自己的文件。

## 一条兼容当前版本的标准示例

```csv
employee_id,name,department,job_title,level,city,contract_type,hire_date,manager
E0001,张三,销售与市场,销售经理,P4,上海,正式员工,2024-03-01,李四
```

## 这一章必须做到什么程度才算完成

只有当导入后下面这些都同步变化，才算真的完成：

- 人员数量
- 部门结构
- 组织总览
- 下钻视图
- 结构信号
- 导出结果
