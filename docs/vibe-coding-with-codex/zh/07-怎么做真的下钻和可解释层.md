# 第 07 章：怎么做真的下钻和可解释层

## “下钻”到底不能假在哪里

很多原型会把“下钻”做成非常表面的效果，比如：

- 只是标题变了
- 只是当前 tab 高亮了
- 只是列表换了几行

但这种不算真正的下钻。

这次项目里，下钻应该意味着：当前部门上下文会影响整个页面系统。

## 切换部门以后，哪些东西应该一起变

当用户选中不同部门时，至少这些内容都应该联动：

- 当前人数
- 管理跨度
- 新人占比
- 人员清单
- 职级结构
- 开放岗位
- 结构信号
- 导出内容

如果这些没动，那就只是视觉切换，不是产品行为。

## “可解释”为什么重要

图里的关键词之一是“可解释”。这意味着系统不能停留在“给你一张图”。

它还需要把图变成一句人能读懂的话，例如：

- 新人占比偏高，说明团队处在扩张期
- 管理跨度偏大，中层承压明显
- 高阶人才占比偏低，说明梯队可能偏薄
- 一线城市集中度过高，说明配置弹性偏弱

有了这层，图表才会真正进入决策语言。

## 可解释层靠什么实现

最简单有效的方式，其实是前端规则：

- 如果新人占比超过某阈值，就生成增长信号
- 如果管理跨度过大，就生成管理负载信号
- 如果高阶占比过低，就生成能力断层信号

规则不一定复杂，但一定要明确。因为没有规则，就没有解释。

## 为什么这一步会让产品感大幅提升

因为它把界面从“展示信息”推进到了“帮助判断”。

很多人一开始以为自己缺的是炫图，其实真正缺的是“这个系统帮我得出了什么结论”。

## 这一章的结论

下钻让产品能被浏览，可解释层让产品能被理解。这两层加起来，才接近“组织地图”真正该有的味道。
